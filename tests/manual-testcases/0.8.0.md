# Manual Test Cases — 0.8.0

## Protocol Modes

### Dual-instance deployment with different protocol modes

**Why manual**: Requires two Gateway listeners, two MCPGatewayExtension resources in different namespaces, and verifying both protocol paths work simultaneously. Infrastructure complexity beyond e2e scope.

- [ ] Deploy Gateway with two listeners (e.g. `mcp-legacy` on port 8080, `mcp-2026` on port 8081)
- [ ] Create MCPGatewayExtension in `mcp-stateful` namespace targeting `mcp-legacy` listener with `protocolMode: Stateful`
- [ ] Create MCPGatewayExtension in `mcp-stateless` namespace targeting `mcp-2026` listener with `protocolMode: Stateless`
- [ ] Register the same MCP server in both namespaces
- [ ] Verify tool call via stateful gateway (port 8080) uses session-based routing
- [ ] Verify tool call via stateless gateway (port 8081) uses header-based routing (`mcp-method`/`mcp-name` headers)
- [ ] Verify both instances return correct tool results independently
